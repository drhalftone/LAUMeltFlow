\begin{abstract}
This report summarizes experiments conducted to develop a neural network surrogate model for predicting numerical flux in compressible Euler equations. We analyze the performance of various architectural choices and conservation constraints on the Sod shock tube problem. Our key finding is that the primary source of error is training data imbalance in extreme boundary states. We address this through uniform grid sampling of the state space, achieving sub-0.5\% relative errors on flux prediction in 1D and approximately 1.3\% in 2D. For 1D, we achieve stable simulations with mean absolute errors of 0.004 kg/m$^3$ for density, 2.0 m/s for velocity, and 330 Pa for pressure. We extend the approach to 2D by concatenating 4 neighbors (plus center cell) to predict all interface fluxes simultaneously, demonstrating successful shock tube simulations with MAE of 0.002 kg/m$^3$ (density), 0.7 m/s (velocity), and 154 Pa (pressure). Finally, we demonstrate that including the specific heat ratio $\gamma$ as an input feature enables a single model to generalize across different gases ($\gamma = 1.2$--1.7), achieving $<$0.2\% relative error on interpolated $\gamma$ values never seen during training.
\end{abstract}

\section{Introduction}

Computational fluid dynamics (CFD) simulations of compressible flows require solving the Euler equations, which describe conservation of mass, momentum, and energy. Traditional numerical methods like the Roe scheme compute numerical flux at cell interfaces to update the solution. We investigate whether a Graph Neural Network can learn this flux function from simulation data.

The test problem is the 1D Sod shock tube, which features:
\begin{itemize}
    \item An expansion fan (smooth rarefaction wave)
    \item A contact discontinuity
    \item A shock wave
\end{itemize}

\section{Methodology}

\subsection{GNN Architecture}

The GNN operates on a graph representation of the finite volume grid:
\begin{itemize}
    \item \textbf{Nodes}: Grid cells with features $[\rho, u, p, \phi, x]$
    \item \textbf{Edges}: Cell interfaces with features $[dx, \text{normal}]$
\end{itemize}

The core component is a FluxMLP that predicts the numerical flux $F^*$ from left and right states:
\begin{equation}
    F^* = \text{FluxMLP}(U_L, U_R, \text{edge\_attr})
\end{equation}

\subsection{Finer Grid (400 Nodes)}

Attempting 4$\times$ finer resolution failed due to the Gibbs phenomenon:
\begin{itemize}
    \item More edges around the shock = more extreme flux gradients
    \item Fewer total samples due to CFL-constrained timestep
    \item Simulation became unstable
\end{itemize}

\section{Solution: Uniform Grid Sampling}

Based on the analysis showing that training data imbalance was the primary bottleneck, we developed a new approach: instead of sampling from simulation trajectories, we sample the state space uniformly.

\subsection{Key Insight: Path Graphs Don't Need Message Passing}

For 1D problems, the computational graph is a path graph where every interior node has exactly one left and one right neighbor. This means we can replace GNN message passing with simple concatenation:

\begin{equation}
    F^* = \text{MLP}([\rho_L, u_L, p_L, \rho_R, u_R, p_R])
\end{equation}

This eliminates the need for PyTorch Geometric and scatter operations, making training faster and simpler.

\subsection{Uniform Grid Sampling}

We define parameter ranges based on the Sod shock tube problem:
\begin{itemize}
    \item Density $\rho$: [0.05, 1.2] kg/m$^3$
    \item Velocity $u$: [-150, 350] m/s
    \item Pressure $p$: [5,000, 120,000] Pa
\end{itemize}

With 10 samples per dimension across the 6D input space (left and right states), we generate $10^6 = 1{,}000{,}000$ training samples with analytically computed Roe flux for each.

\section{Extension to 2D}

Building on the success of the 1D MLP flux predictor, we extend the approach to 2D problems. The key insight is that instead of computing individual interface fluxes, we can use a single MLP that takes all 4 neighbors (plus center cell) and predicts all 4 interface fluxes simultaneously.

\subsection{2D Architecture}

For 2D Euler equations, the conserved variables are $[\rho, \rho u, \rho v, E]$ (4 components). The MLP architecture is:

\begin{equation}
    [F_w, F_e, G_s, G_n] = \text{MLP}([\text{W}, \text{C}, \text{E}, \text{S}, \text{N}])
\end{equation}

where W, C, E, S, N are the West, Center, East, South, and North cell primitive variables $[\rho, u, v, p]$.

\begin{itemize}
    \item \textbf{Input}: 20 features (5 cells $\times$ 4 primitives)
    \item \textbf{Output}: 16 fluxes (4 faces $\times$ 4 flux components)
    \item \textbf{Architecture}: MLP with 256$\times$5 hidden layers (272,656 parameters)
\end{itemize}

\subsection{Training Data Generation}

Unlike 1D where we could use a uniform grid ($10^6$ samples), 2D has 20 input dimensions making grid sampling infeasible ($n^{20}$ would be astronomical). Instead, we use random uniform sampling within the parameter ranges:

\section{Generalization to Different Gases: Varying $\gamma$}

A key limitation of the previous models is that they were trained with a fixed specific heat ratio $\gamma = 1.4$ (appropriate for air and diatomic gases). To create a more general flux predictor applicable to different gases, we extended the model to accept $\gamma$ as an input feature.

\subsection{Motivation}

The specific heat ratio $\gamma = c_p/c_v$ varies significantly across gases:
\begin{itemize}
    \item \textbf{Monatomic gases} (He, Ar, Ne): $\gamma = 1.67$
    \item \textbf{Diatomic gases} (air, N$_2$, O$_2$): $\gamma = 1.4$
    \item \textbf{Triatomic gases} (CO$_2$, H$_2$O vapor): $\gamma \approx 1.3$
    \item \textbf{Complex molecules, combustion products}: $\gamma = 1.1$--$1.2$
\end{itemize}

A model that learns the parametric dependence on $\gamma$ can generalize across this entire range without retraining.

\subsection{Training with $\gamma$ as Input}

We modified the flux predictor to accept 7 inputs instead of 6:
\begin{equation}
    F^* = \text{MLP}([\rho_L, u_L, p_L, \rho_R, u_R, p_R, \gamma])
\end{equation}

Training data was generated by sampling 10 $\gamma$ values uniformly from 1.2 to 1.7, combined with the $10^6$ state space samples, yielding \textbf{10 million training samples}.

\subsection{Why Including $\gamma$ as Input Improves Results}

This $\gamma$-parameterized model achieves \textbf{better results than the fixed-$\gamma$ model} from earlier experiments. The key reasons are:

\begin{enumerate}
    \item \textbf{10$\times$ more training data}: 10 million samples vs 1 million
    \item \textbf{Explicit physics encoding}: The network learns how $\gamma$ enters the equation of state $E = p/(\gamma-1) + \frac{1}{2}\rho u^2$ and the Roe flux computation
    \item \textbf{Smooth interpolation}: Because $\gamma$ is a continuous input, the network learns a smooth function that interpolates naturally between training values
    \item \textbf{Richer training signal}: Seeing the same $(\rho, u, p)$ states with different $\gamma$ values teaches the network the parametric dependence, not just memorization
\end{enumerate}

This demonstrates the principle of \textbf{physics-informed machine learning}: including known physical parameters as explicit inputs (rather than hardcoding them) leads to better generalization.

\section{Conclusions}

\begin{enumerate}
    \item \textbf{Training data imbalance was the primary bottleneck} -- uniform grid sampling of the 6D state space resolves this issue completely
    \item \textbf{Path graphs don't need message passing} -- simple MLP with concatenated left/right states works well for 1D problems, eliminating the need for PyTorch Geometric
    \item \textbf{Standard MSE loss works best} -- no conservation constraints needed; they all degraded performance
    \item \textbf{GPU training is essential for scale} -- 1000 epochs in 170 seconds vs 17 minutes for 50 epochs on CPU
    \item \textbf{Sub-0.5\% relative errors achieved in 1D} -- 0.46--0.51\% on flux prediction, MAE of 0.004 kg/m$^3$ (density), 2.0 m/s (velocity), 330 Pa (pressure) in simulation
    \item \textbf{Model size matters} -- the 256$\times$5 baseline (266K params) is optimal; larger models overfit despite lower training loss
    \item \textbf{Analytical solver is still faster for small problems} -- the MLP approach is 5$\times$ slower than vectorized Roe for 100-cell simulations due to GPU overhead
    \item \textbf{2D extension successful} -- the 5-cell stencil approach (4 neighbors + center) achieves 1.27\% relative error on flux prediction and produces stable, accurate simulations
    \item \textbf{Random sampling works for high-dimensional inputs} -- when grid sampling becomes infeasible (20D input space), random uniform sampling provides adequate coverage
    \item \textbf{Physics parameters as inputs enable generalization} -- including $\gamma$ as the 7th input allows a single model to handle gases from $\gamma = 1.2$ to 1.7, with $<$0.2\% relative error even on interpolated values never seen during training
\end{enumerate}

\section{Conclusions}

\begin{enumerate}
    \item \textbf{Training data imbalance was the primary bottleneck} -- uniform grid sampling of the 6D state space resolves this issue completely
    \item \textbf{Path graphs don't need message passing} -- simple MLP with concatenated left/right states works well for 1D problems, eliminating the need for PyTorch Geometric
    \item \textbf{Standard MSE loss works best} -- no conservation constraints needed; they all degraded performance
    \item \textbf{GPU training is essential for scale} -- 1000 epochs on 20M samples in 47 minutes
    \item \textbf{Sub-0.1\% relative errors achieved} -- 0.02--0.03\% on flux prediction with the multiphase model
    \item \textbf{Model size matters} -- the 256$\times$5 baseline (266K params) is optimal; larger models overfit despite lower training loss
    \item \textbf{2D extension successful} -- the 5-cell stencil approach (4 neighbors + center) achieves 1.27\% relative error on flux prediction and produces stable, accurate simulations
    \item \textbf{Physics parameters as inputs enable generalization} -- including $\gamma$ as the 7th input allows a single model to handle gases from $\gamma = 1.2$ to 1.67
    \item \textbf{Importance sampling improves coverage} -- combining core region (60\%) and extended region (40\%) sampling ensures both accuracy and robustness
    \item \textbf{Two-fluid interfaces handled correctly} -- the model generalizes to problems with spatially varying $\gamma$
    \item \textbf{Smaller models viable for production} -- a 3$\times$128 architecture (34K params) achieves similar accuracy to the 5$\times$256 baseline (266K params) while being 7.7$\times$ smaller and 3.6$\times$ faster to train
\end{enumerate}

\section{Future Work}

\begin{itemize}
    \item \textbf{3D extension}: Apply the approach to 3D Euler equations with 6-neighbor stencil
    \item \textbf{Real gas effects}: Include temperature-dependent $\gamma$ and other thermodynamic properties
    \item \textbf{Viscous fluxes}: Extend to Navier-Stokes equations with viscosity and heat conduction
    \item \textbf{Multi-species}: Train on problems with species transport and chemical reactions
    \item \textbf{Adaptive refinement}: Use model uncertainty to guide mesh adaptation
    \item \textbf{Hybrid solver}: Use MLP for bulk computation, analytical Roe near discontinuities
\end{itemize}
